
데이터 전달 방식 2가지( default GET)
GET 주소창으로 전달 			보안에 취약, 대용량 불가. 속도가 빠름.

POST 파일 내부로 심어서 전달.  보안에 우수, 대용량 가능. 속도가 상대적으로 느림.

GET 방식은 주소창에 사용자가 입력한 값들이 그대로 노출되는 것을 볼수 있다.
패스워드 같은 보안이 필요한 값들 까지 노출이 되면 문제가 된다.
POST 방식을 사용하면 이러한 문제점을 해결 할 수 있다.


servlet 서블렛은 Server Applet 의 약자로 자바 기반의 동적 웹 프로그래밍 솔루션이다.
서블렛을 기반으로 JSP가 개발 되었다. JSP 내부적으로는 서블릿으로 변환되어 수행된다. 따라서 
서블렛의 구성, 동작방식을 이해하면 JSP를 이해하는데도 많은 도움이 된다.

JSP로 구현해 보자.
JSP Java servlet page. JSP는 서블렛에 비해 쉬운 프로그래밍 방법을 제공한다. 하지만 내부적으로는 서블렛으로 변화하여
실행된다.
